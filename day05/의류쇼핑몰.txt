- 회원가입
- 로그인
- 제품 상세
- 제품 등록
- 제품 수정
- 제품 구매

<요구사항명세서>
1. 회원가입
- 아이디, 비밀번호, 이름, 전화번호를 입력하면 회원가입이 된다
- 아이디는 중복될 수 없다
- 한 사람당 하나의 아이디만 갖는다 -> 전화번호로 판별

2. 로그인
- 아이디와 비밀번호를 입력하면 아이디와 비밀번호가 일치하면 로그인된다
- 일치하지 않으면 로그인되지 않고 재시도한다. 최대 5번
- 로그인 시도 5번 초과시 해당 아이디는 보호된다. 

3. 제품 상세
- 쇼핑몰에서 소유한 제품을 보여준다
- 구분(외투, 상의, 바지, 치마, 원피스), 제품명, 가격, 색상, 사이즈를 보여준다.
- 전체, 구분별로 제품을 볼 수 있다.
- 낮은가격순, 높은가격순, 판매량순으로 볼 수 있다.

4. 제품 등록
- 구분, 제품명, 수량, 가격, 색상, 사이즈를 입력하면 등록된다.
- 구분과 제품명이 똑같이 일치하는 제품은 없다
- 일치하는 제품이 있으면 있다고 알려주고 다시 입력하게 한다

5. 제품 수정
- 구분, 제품명을 입력하면 일치하는 제품이 있으면 수량, 가격, 색상, 사이즈를 수정할 수 있다
- 일치하는 제품이 없으면 없다고 알려주고 다시 입력하게 한다

6. 제품 구매
- 구매를 원하는 제품의 구분과 제품명을 입력하면 일치하는 제품이 있으면 사이즈, 색상, 수량을 입력한다.
- 일치하지 않으면 해당 제품은 없다고 알려주고 다시 입력하게 한다.
- 장바구니에 담기고 일괄적으로 장바구니에 담긴 제품들을 구매할거냐고 묻고 구매한다고 하면 구매됨
- 구매 안한다고 하면 메뉴로 돌아감

<ERD 설계>
- 회원 : 아이디, 비밀번호, 이름, 생년월일, 주소
- 제품 : 제품 타입(상의, 하의, 원피스, 신발), 제품명, 제품코드, 제품 설명, 사이즈, 제품 크기별 구매 가능 수량, 가격

<선생님 요구사항 명세서>
1. 회원가입
- 회원가입 시 아이디, 비밀번호, 비밀번호 확인, 생년월일, 주소를 입력한다
- 일반 회원과 관리자는 권한만 다르고 나머지 정보는 같다.
- 관리자는 미리 등록된 회원 정보를 이용한다.(아이디/비밀번호 : admin/admin)
- 아이디, 비번, 비번확인은 공백없이 입력한다.
- 생년월일은 yyyy-MM-DD 형태로 입력
- 주소는 공백을 포함한 한줄을 입력한다.
- 일반 회원은 권한이 USER이고, 관리자는 권한이 ADMIN이다
 
2. 로그인
- 아이디와 비번을 입력하여 로그인한다
- 일치하는 회원 정보가 없으면 로그인 실패, 있으면 로그인 성공

3. 제품 상세
- 제품에 대한 정보를 보여준다
- 제품 타입(상의, 하의, 원피스, 신발), 제품명, 제품코드, 제품 설명, 사이즈, 제품 크기별 구매 가능 수량, 가격 보여줌 

4. 제품 등록
- 제품 정보를 입력하여 제품 등록
- 제품 타입, 제품명, 제품코드, 제품설명, 사이즈, 제품 크기별 구매 가능 수량, 가격 입력
- 제품은 관리지만 등록한다.

5. 제품 수정
- 제품 정보를 입력하여 제품 수정
- 제품 타입, 제품명, 제품코드, 제품설명, 사이즈, 제품 크기별 구매 가능 수량, 가격 중 수정하고 싶은 내용을 선택하여 수정한다.
- 제품은 관리지만 수정한다.

6. 제품 구매
- 제품 상세에서만 할 수 있다
- 제품은 모든 회원이 구매할 수 있다
- 제품을 선택, 제품 크기와 수량을 선택하여 구매한다
- 제품을 구매하기 위해 배송지를 입력한다. 
- 제품 구매시 한번에 한 종류만 구매 가능하다.

회원
아이디 : varchar(20)
비번 : varchar(16)
생년월일 : date
주소 : varchar(100)
권한: varchar(5)
